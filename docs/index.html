Try the Silent Failure Test

A generic tool reports the CloudTrail event closest in time to an S3 cost spike. But correlation isn't causation. Can you spot which event actually caused the spike?

S3 bill spiked +$1,840 on 2024-01-18  ·  Two events found in CloudTrail ±24 h of the spike  ·  Which one caused the cost increase?
PutBucketPolicy
Applied a new bucket policy restricting public read access.
Logged 3 hours before the spike.
PutBucketLifecycleConfiguration
Removed all lifecycle rules that moved objects to S3-IA after 30 days.
Logged 6 hours before the spike.
✗ Not quite — that's a silent failure
PutBucketPolicy changes who can access a bucket — it controls IAM permissions, not storage pricing. It cannot cause a cost spike. A naive system flags it because it's the nearest event in time, but CostSherlock rules it out as WRONG_MECHANISM — access policies have no effect on S3-Standard storage pricing.
✓ Correct — that's the real cause
PutBucketLifecycleConfiguration deleted the rule that was automatically moving objects to the cheaper S3-IA tier after 30 days. With the rule gone, all objects stayed in S3-Standard — the more expensive tier — and the bill reflected the difference. CostSherlock's Analyst validates this with unit math: 450 GB × ($0.023 − $0.0125) / GB = $4.73/day ≈ $1,840/month.
Key Features

Built for Rigor,
Not Just Speed

Every design decision prioritizes evidence quality and causal correctness over fast, hand-wavy answers.

🤖

Multi-Agent Investigation

Four specialized AI agents — Sentinel, Detective, Analyst, Narrator — each with a single responsibility, chained in sequence. No agent skips a step or does another's job.

📚

RAG-Powered Reasoning

50+ curated AWS pricing and troubleshooting documents stored in ChromaDB with local MiniLM embeddings. Every hypothesis is grounded in retrieved pricing evidence — zero hallucinated costs.

🎯

Silent Failure Detection

Catches correlation-without-causation errors automatically. A PutBucketPolicy event near an S3 cost spike? Ruled out as WRONG_MECHANISM — it changes access permissions, not storage pricing.

🖥️

Interactive Dashboard

5-view Streamlit interface: Timeline with anomaly severity table, Investigation report viewer with Previous/Next navigation, Evidence Explorer, cross-anomaly Compare charts, and a Feedback loop.

📎

Evidence-Grounded Reports

Every factual claim in every report carries an inline citation — [CloudTrail: RunInstances], [Pricing: ec2_ondemand_pricing.md]. Conclusions beyond evidence are tagged [INFERENCE].

📊

Evaluation Framework

7 metrics including causal attribution accuracy, evidence recall, RAGAS faithfulness scoring, time-to-explanation, human audit pass rate, and a feedback loop quality methodology.

The Dashboard

The Investigation Dashboard

A fully interactive Streamlit application with five purpose-built views for exploring, navigating, and auditing cost investigations.

1

Timeline

Interactive Plotly cost trend chart, 4 KPI metric cards, and a sortable anomaly table with one-click "Investigate" buttons that auto-run the full pipeline.

2

Investigation

Full 7-section report rendered in markdown with Previous / Next navigation across all investigated anomalies. Colour-coded confidence levels and status banners.

3

Evidence Explorer

Per-hypothesis expandable cards showing evidence items, quantitative cost calculations, causal mechanism descriptions, and ruled-out events with reasons.

4

Compare

Grouped Plotly bar chart overlaying expected vs. actual costs across all anomalies simultaneously. Side-by-side confidence scores for quick triage.

5

Feedback

Human audit interface: actionability rating, free-text actual root cause field. Submissions save timestamped JSON files that can be ingested back into ChromaDB to improve future investigations.

Design Rationale

Why Not Just Ask ChatGPT?

A generic LLM has no access to your actual cost data, your CloudTrail events, or your real AWS pricing. Here's what separates a multi-agent system from a prompt.

Generic LLM

"Why did my AWS bill spike?"

  • ❌ Guesses based on training data — no access to your actual costs
  • ❌ Cannot read your CloudTrail event logs
  • ❌ No quantitative cost math — confabulates plausible-sounding numbers
  • ❌ Cannot distinguish correlation from causation in your specific context
  • ❌ No audit trail — you cannot verify where claims came from
  • ❌ One-shot: cannot explore different hypotheses interactively
CostSherlock

Agentic investigation on your real data

  • ✅ Reads your actual Cost Explorer JSON — real numbers, not guesses
  • ✅ Correlates against your real CloudTrail event logs
  • ✅ Validates every hypothesis with unit × price = cost math
  • ✅ Explicitly rules out wrong mechanisms before attributing cause
  • ✅ Every claim cites a specific event ID or pricing document
  • ✅ Interactive: explore, compare, and provide corrective feedback

The key insight: cost anomaly investigation requires real data access, multi-step orchestration (detect → correlate → reason → report), quantitative validation (does the math actually work out?), and evidence grounding (where did this claim come from?). A single LLM call can do none of these reliably. A four-agent pipeline, by design, does all of them.
